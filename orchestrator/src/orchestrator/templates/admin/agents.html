Agents represent security software (EDR, AV, DLP, etc.) installed on target servers. Each agent has Detection Rules that define how to verify if the agent is installed on a given OS (e.g., check for a service name, run a version command). Detection rules are used during snapshot creation to automatically verify agent installation and capture version information.

Detection Rules
Detection rules define how to check if this agent is installed on a server, per OS type. OS Regex matches against the server's OS vendor (e.g., "rhel|rocky" or "windows"). Service Regex is used to check if the agent's service is running. Version Command is run via SSH to capture the installed version.
